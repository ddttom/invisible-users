- [ ] Claude skills static snapshot blog post
  - Draft location: [docs/structure/blog-drafts/claude-skills-static-not-subroutines.md](blog-drafts/claude-skills-static-not-subroutines.md)
  - Status: Draft created 26/Jan/2026
  - Key concept: Skills are static snapshots locked into time at creation, not dynamic subroutines
  - **Summary:** Claude Code skills are NOT subroutines that execute dynamically. When a skill is created, it reads the authoritative source (e.g., a template file, documentation), extracts patterns, and hard-codes instructions into the skill.md file. If the authoritative source changes later, the skill does NOT understand those changes - it remains locked into the state from creation time. Skills must be manually regenerated to incorporate updated source material. Common misconception: developers expect skills to re-read sources on each invocation (they don't). This is an architectural constraint, not a bug.
  - Questions to resolve:
    - Should this concept be added to MX-Bible book?
    - Where in the book? (Chapter 12 Technical Advice? New appendix on AI tooling?)
    - Should we create a paragraph or two documenting this architectural principle?
    - Does this relate to the broader MX pattern of "documentation as static snapshot vs dynamic reference"?
  - Next steps: Review for technical accuracy, decide on book integration
