The Solution

Make Implicit State Explicit

You don't need to rebuild everything. You need to make your excellent design machine-readable.

❌ What Breaks Agents

<button className={isValid ? 'active' : 'inactive'}>
  Submit
</button>

Button colour changes via CSS. State is invisible to agents.

→

✅ What Works for Everyone

<button
  className={isValid ? 'active' : 'inactive'}
  disabled={!isValid}
  data-form-state={isValid ? 'complete' : 'incomplete'}>
  Submit
</button>

Same design. State is explicit in the DOM. Works for agents AND screen readers.

Five Core Principles

1

Clear State

Use HTML attributes, not just CSS classes

2

Persistent Errors

No toast notifications - errors stay visible

3

Complete Information

Semantic Structure

5

Explicit Feedback

Confirm actions clearly in page content

FAQ

Common Questions

Is this book for me if I'm not a developer?

Yes. The book has specific reading paths for business leaders, product managers, designers, and content creators. Chapters 4-8 focus on business implications with minimal technical jargon. You can skip or skim the implementation code in Chapter 10.

How technical is the content?

Chapters 1-9 are accessible to anyone working with websites. Chapter 10 contains production-ready code examples in HTML, JavaScript, React, and WordPress. Code is clearly marked and optional for non-developers.

Will this become outdated quickly?

The core principles (semantic HTML, clear state, persistent feedback) are timeless. The specific agent behaviours may evolve, but the book focuses on fundamental design patterns that work regardless of AI platform. You'll receive lifetime updates.

Do I need to support AI agents if I have low traffic?

Chapter 4 includes a decision tree. Local/relationship-based businesses can often deprioritise this. E-commerce, SaaS, content publishers, and price-competitive businesses should act now. The book helps you assess your specific situation.

Can I implement these patterns on my existing site?

Yes. The book emphasises incremental improvements over rebuilding. Many fixes take minutes or hours, not weeks. Chapter 9 provides a priority-based implementation roadmap.

What frameworks/platforms are covered?

The principles apply universally. Code examples include vanilla HTML/JavaScript, React, Next.js, WordPress, and static site generators. The Agent-Friendly Starter Kit includes both good and bad implementations you can run locally.
