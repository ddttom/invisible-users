Universal Identity Delegation Infrastructure

Open-source, portable agent authorization that preserves user control and customer relationships

⚠️ Project Status: Planning Phase

This project is currently in the planning and design phase. The proof-of-concept code from "The Invisible Users" book demonstrates the core concepts. A production-ready implementation will be developed as an open-source project with community collaboration.

The Problem

Every major platform is building closed identity systems for AI agents. Microsoft, Google, Anthropic, and OpenAI are racing to establish proprietary delegation mechanisms. When you authorize an agent through one platform, you're locked into that ecosystem. Switch agents? Re-authorize everything. The customer relationship is trapped.

As described in The Invisible Users Chapter 11, this creates:

  • User lock-in: Switching agents means losing all delegated access
  • Business loss: Merchants lose customer identity when agents make purchases
  • Agent fragmentation: Each platform builds incompatible systems
  • Trust erosion: Users can't control or audit agent authorizations

The Solution

A universal identity delegation infrastructure that enables:

For Users

Grant and revoke agent access from one place. Your authorization works across platforms. See all active delegations in one dashboard.

For Agents

Standard delegation tokens work everywhere. No platform-specific integration needed. Portable authorization that moves with users.

For Businesses

Preserve customer identity through agent-mediated purchases. Loyalty programs work. Order history maintained. Marketing effective.

For the Ecosystem

Open standards prevent platform lock-in. Community governance ensures interoperability. Users control their data.

Technical Approach

The infrastructure will provide:

Core Components

  • Delegation Token Provider: OAuth 2.0 extension for portable authorization tokens
  • Token Verification Service: Cryptographic validation for businesses
  • Revocation Registry: Fast, distributed token revocation checking
  • User Dashboard: Manage all active agent authorizations
  • Agent SDK: Libraries for token generation and validation

Implementation Libraries

  • JavaScript/Node.js (Cloudflare Workers, Express middleware)
  • Python (FastAPI, Django middleware)
  • Go (standard library HTTP handlers)
  • Ruby (Rails middleware)

Standards Compliance

  • OAuth 2.0 delegation extension
  • JWT tokens with standard claims
  • OpenID Connect compatibility
  • GDPR-compliant data handling

How It Works

The delegation flow enables portable, user-controlled authorization:

  1. User authorizes agent: Grant permission through delegation provider dashboard
  2. Agent receives token: Portable JWT token with standard claims (sub, agent_id, scope, exp)
  3. Agent acts on behalf: Include delegation token in requests to businesses
  4. Business verifies token: Cryptographic verification preserves customer identity
  5. User maintains control: Revoke delegation at any time from centralized dashboard

Unlike proprietary systems, delegation tokens work across all compliant agents and merchants. Authorization is portable. Users remain in control.

Why Open Source?

Identity delegation is too important to be controlled by any single platform. Users should control their authorizations, not platforms. Open governance ensures:

  • User agency: Delete authorizations without platform permission
  • Interoperability: Community-driven standards prevent fragmentation
  • Transparency: Open code enables security auditing
  • Longevity: No single company can abandon the project

This infrastructure belongs to the ecosystem, not to any vendor.
